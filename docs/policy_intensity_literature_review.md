# 房地产政策强度测量文献综述

本文件记录 2026-07-22 能够核验到的原文方法。它不是“根据题名猜算法”的文献列表：仅能取得摘要的研究只用于界定概念，不据此复刻未公开的公式。

## 可复用的核心结论

1. **政策密度与政策强度必须分开。** Schaffrin、Sewerin 与 Seubert（2015）将政策产出拆成密度与强度，并从目标、范围、整合、预算、实施和监测六个设计要素编码强度。Schmidt 与 Sewerin（2019）进一步采用自下而上的工具级编码，再聚合为政策组合强度，并使用多人编码和争议裁决。这支持本项目从“政策动作”而不是整篇文件直接打分。
2. **政策动作方向与工具校准必须分开。** IMF iMaPP（Alam et al., 2019）按月记录 17 类宏观审慎工具的收紧/放松动作，并对 LTV 等可量化工具保存水平信息。这支持分别保存方向、动作数量和旧值/新值，不能用一个总分替代。
3. **文本模型需要可靠人工标签和分组外推验证。** Gentzkow、Kelly 与 Taddy（2019）强调预处理本身具有实质含义，惩罚线性模型是高维文本的稳健基线，外样本漂移和不确定性必须报告。Wang 等的规划政策 NLP 研究使用专家编码句子、训练/验证/测试集，并比较 BERT 与不同提示方式的大模型。因此本项目保留 TF-IDF 线性基线，Transformer 和 GLM 均不是唯一真值来源。
4. **机器学习可提升规模，但不能消除测量设计。** Xie et al.（2024）可核验的摘要显示，其程序先给政策措施维度打分，再与政策标题、文件类型维度交叉计算。由于公开摘要不足以复刻全部训练和计分细节，本项目不复制其未核验公式，只采纳“维度分解、模型辅助、最终确定性聚合”的原则。
5. **中文政策量化必须提供句子计数、人工检查和代码。** Zhen、Li 与 Wang（2026）公开方法包括句子切分、规则识别政策工具、人工标注目标、向量化器与分类器比较、分层抽样人工复核，并公开原始句子计数和代码。其工具优先级与 3/2/1 分值具有特定碳政策语境，本项目不直接移植权重。
6. **房地产工具需要领域校准。** 王羲泽等（2023）以 49 个城市的住房限购强度研究政策扩散，说明城市级限购规则可以被量化并用于时空研究；但房地产数据库还包含贷款、公积金、补贴、保障房、城市更新等异质工具，因此必须逐工具定义旧值、新值、方向和标准化变化。

## 对本项目的直接约束

- 测量对象是文本中有证据的政策设计、实施承诺和工具校准，不是政策执行效果。
- 最小单元为政策动作；文件、版本、条款、证据区间与动作保持可追溯关系。
- 数值事实由确定性解析优先；GLM 只能辅助复杂语义并给出证据。
- `NA` 表示维度不适用，聚合时重归一权重；不能把 `NA` 当作 0。
- 正式研究分数仅允许完整或经人工确认足够的官方正文。摘要、媒体转载和残缺文本只能生成 provisional 结果。
- 传统机器学习、Transformer、GLM 和规则分别保存预测；融合路由器保存接受值、理由、一致性和置信度。
- 模型性能必须按政策实体/转载族分组切分，并增加城市和时间留出；不能让同一文件转载泄漏到训练集和测试集。
