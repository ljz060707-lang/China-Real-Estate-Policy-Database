## SiliconFlow 验收

- 已实现统一 `AIProvider`、OpenAI-compatible 客户端、Keyring/环境变量密钥读取、模型列表、
  结构化分类、独立复核、Embedding 和 Rerank 接口。
- 动作分类必须包含原文中唯一匹配的证据、开始偏移和结束偏移；不匹配或不唯一的证据会被拒绝。
- 实际 `/v1/models` 连接返回 HTTP 401。现存旧 GLM 兼容密钥不是有效 SiliconFlow 凭据。
- 分类模型和独立复核模型未配置，因此本次没有执行真实付费分类或复核。
- 自动分类率、macro-F1 和自动审核通过率没有真实评估，不作研究质量声明。

配置方法见 `docs/siliconflow_ai_provider_guide.md`。必须由用户在本地保存有效 Key，并从模型列表
中明确选择分类和复核模型后，才可以运行真实批次。

## 归档验收

实际执行 `policydb archive sync` 和 `policydb archive audit`。档案根目录使用
`D:\Data Set\CRPD`；181 个文档版本完成内容寻址归档和 SHA-256 复核，100 个版本因源文件不存在
保留 `missing_source_file`。命令重复运行保持幂等，不覆盖历史 Raw。

当前 181/281（64.41%）是“文档版本归档率”，不能替代全部政策记录的全文归档率，也未达到
95% 的目标。

## 去重验收

已将政策实体、发布版本和重复簇分开保存。规则使用规范 URL、内容哈希、标题、机关、日期及文本
相似度；不会因标题相同直接合并不同地区政策。30 个候选簇中，25 个满足确定性精确规则，5 个
需要复核；删除记录数为 0。

SiliconFlow 语义去重接口已经接入，但因认证失败未执行真实模型裁决。

## 105 城完整性验收

已生成 105 城 × 4 类核心来源 × 103 个月的 43,260 个覆盖单元。四类来源为政府门户、住建、
公积金和自然资源。当前严格口径下没有任何完整来源窗口，因此全部 43,260 个单元保留为缺口。

这不表示数据库没有地方政策，而表示来源注册、分页扫描和可复核扫描证据尚不足以确认完整性。
系统不会把 `unscanned`、`partial` 或 `failed` 写成零政策。

## Windows 调度验收

每日、周度和月度任务定义、状态查询、安装、删除和网站入口已实现。实际状态检查显示三类任务均
为 `not_installed`；本次没有在用户机器上擅自安装计划任务。安装和删除都要求显式 `--confirm`。

## Dashboard 浏览器验收

本地 Streamlit 健康检查返回 HTTP 200。通过浏览器实际打开并检查：

- 数据总览：3,568 条政策、最新日期 2026-07-02、待审核 22；
- 政策中心：4,337 个动作行，含五域筛选、证据、原文和档案区；
- 自动更新与完整性：43,260 个覆盖单元、0 个完整窗口、43,260 个缺口，并明确显示 Windows
  计划未完整安装。

页面无顶层报错；一次页面切换调用返回浏览器控制层 `Internal error`，随后的 DOM 快照确认页面
已正确切换并完整渲染，不是 Streamlit 页面异常。

## 研究使用边界

以下状态仍为否：

- `research_ready=false`：尚无经裁决的人工金标准和独立留出集；
- 真实 SiliconFlow 模型质量未评估；
- 105 城 2018 年以来完整覆盖未实现；
- 文档版本归档率未达到 95%；
- Windows 计划任务未实际安装。

在这些门槛满足前，模型输出、语义去重和政策强度只能作为暂定研究辅助信息，不能直接作为正式
论文结论。
