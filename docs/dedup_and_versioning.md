# 去重与版本识别

系统沿用现有抓取管线，执行 L0—L7 七层闸门：任务键、规范 URL、HTTP 条件请求、二进制 SHA-256、规范正文 SHA-256、政策身份键、相似版本、GLM 缓存键。每项自动决定写入 `dedup_decisions`，保留规则版本、阈值、比较对象和证据。

URL 规范化删除 fragment 和追踪参数、排序保留参数、统一常见移动端域名，但保留分页、文件 ID 和日期等有事实意义的参数。HTTP 304 不重新下载、解析或调用 GLM。相同二进制或规范正文复用已有版本，同时保留来源出现关系。

相似版本使用 SimHash 与文本相似度辅助判断；标题、机关、日期、文号构成政策身份证据。任何金额、比例、日期、年限或数量冲突均判为 `material_change`，不得因高相似度自动合并。官方原文和转载保留不同网页版本，但可关联同一政策实体。

GLM 缓存键为 `normalized_text_hash + model + prompt_version + schema_version`。模型、提示或 schema 变化会生成新缓存；相同输入不重复付费调用。
