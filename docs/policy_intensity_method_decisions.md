# 政策强度方法决策记录

## 已决定

1. 正式名称为 `textual_policy_design_intensity`、`textual_implementation_commitment_intensity` 和 `instrument_calibration_intensity`。页面中文明确标注“文本测量”，不称为政策效果。
2. 正式最小单元是 `policy_action`。动作保留版本、条款、证据文本和字符偏移。
3. D1–D7 使用 0–3 rubric，D8 使用确定性校准强度。`NA` 不进入分母。
4. 规则抽取的数值事实优先于 GLM；数值冲突进入审核而不是平均。
5. 完整官方正文或人工确认足够正文才产生 formal 分数；其余均为 provisional。
6. 等权是默认可解释主版本。文献权重、专家权重、熵权和 PCA 只能作为并列敏感性版本；PCA 不作为主指标。
7. 权威调整、质量调整与原始文本强度分列，避免把来源质量混进政策设计本身。
8. 中央/省级政策通过适用范围关系聚合，不能复制为 105 条“地方政策”。城市月度表同时输出动作数、覆盖政策数、gross/net 与来源层级构成。

## 当前不能声称完成的事项

- 尚无 500 篇文件、3,000 条款、20% 双人编码的裁决金标准，所有自动强度结果只能标记为 `experimental` 或 `provisional`。
- 未安装 Transformer 训练依赖，未下载中文模型；本阶段只提供可选、懒加载接口。
- 当前未配置可用 GLM Key，不能报告真实 GLM 准确率、费用或吞吐。
- 现有数据库 3,568 条记录中，完整官方正文必须按保守规则重新判定，不能把摘要长度直接当作完整性。
- 正式门槛（动作 macro-F1≥0.85、工具 macro-F1≥0.85、方向 macro-F1≥0.90、数值及配对≥0.95、Kappa≥0.75、指数 Spearman≥0.80）未达到前，系统不得输出“研究就绪”。

## 版本策略

- codebook、动作规则、约束词典、校准标度分别版本化。
- 预测缓存键包含正文哈希、任务、模型、模型版本、提示版本和 schema 版本。
- 路由决策保存路由器版本、候选模型、接受路径和回退原因。
- 修改权重或标度只生成新版本分数，不覆盖历史结果。
