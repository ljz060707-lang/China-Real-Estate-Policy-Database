# 字段证据与置信度方法

`field_confidence` 保存字段值和证据，不把综合分反写主记录。默认确定性权重位于 `config/quality_rules.yml`：来源权威 0.30、证据覆盖 0.25、跨来源一致 0.20、抽取确定性 0.15、实体匹配 0.10。

每个字段同时保存 evidence excerpt、位置、URL、抽取方法、冲突值及 scoring/model/prompt 版本。记录级分数由字段分推导，并对关键字段的最低分和冲突进行惩罚。

- high：通常要求分数不低于 0.85、官方证据且无关键冲突；
- review：0.65 至 0.85 或证据不足；
- hold：低于 0.65、仅有非官方线索或存在关键事实冲突。

模型只能提取和核对原文证据，不能补写原文不存在的事实。最终自动通过由确定性规则决定；低分或冲突进入少量人工复核。规则版本变化后可重算派生分，但历史证据和人工修正必须保留。
